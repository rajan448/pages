Performance Testing Types

Load Testing

Evaluates system behavior under expected load. Aims to determine if the application can handle the anticipated user traffic without performance degradation.

Stress Testing

Tests the system beyond its normal operational capacity to determine its breaking point and how it recovers from extreme loads. Identifies bottlenecks under high stress.

Endurance (Soak) Testing

Checks system stability and performance over a prolonged period (e.g., hours or days) to detect memory leaks, resource consumption issues, and degradation.

Spike Testing

Observes system behavior when the load suddenly and drastically increases or decreases, simulating sudden surges in user activity.

Scalability Testing

Determines the system's ability to scale up or down (e.g., adding more users, data, or hardware) to meet increasing or decreasing performance demands.

Volume Testing

Assesses system performance and behavior when a large amount of data is processed or stored in the database. Focuses on data capacity and efficiency.

Concurrency Testing

Evaluates how the application handles multiple users accessing the same features or data simultaneously. Ensures data integrity and correct processing.
